1. Product Overview
IEP Desk is a free AI-powered web application that helps parents of children with learning
disabilities navigate the American special education system. Parents upload their child's IEP
document and receive a plain English analysis, legal rights breakdown, and a personalized
meeting preparation kit — all powered by AI.
1.1 Problem Statement
7.5 million children in the US have Individualized Education Programs (IEPs). These are legal
documents guaranteeing specialized educational support. However:
• Documents are 20+ pages of dense legal jargon that most parents cannot understand
• Parents attend IEP meetings outnumbered by school specialists and feel powerless
• 40% of IEP parents report feeling excluded from the process legally designed to include
them
• No dedicated SaaS product exists specifically for parents — only tools for schools and
teachers
1.2 Solution
IEP Desk gives parents three core capabilities:
1. Upload IEP document → AI analyzes in plain English, flags red flags and missing
accommodations
2. Enter child diagnosis and state → AI shows exact legal rights and accommodations
entitled to
3. Before any meeting → AI generates personalized questions, pushback scripts, and email
templates
1.3 Target Users
User Type Description Why They Use IEP Desk
Primary Parents Parents of IEP/504 children Understand rights, prepare for
meetings
Secondary Advocates IEP advocates, consultants Tool for client preparation
Tertiary Schools School districts (future) White-label licensing

3. System Architecture
3.1 High Level Architecture
System Flow
User Browser (React) → Vercel CDN → React App
React App → Firebase Auth → Get ID Token
React App → Render.com → Express API (with Firebase token in header)
Express API → MongoDB Atlas → Read/Write data
Express API → OpenRouter API → AI analysis
Express API → Cloudinary → Upload/retrieve files
Express API → Resend.com → Send emails
Express API → React App → Return JSON response
3.2 Authentication Flow
Authentication uses Firebase Auth on the frontend and Firebase Admin SDK on the backend to
verify tokens.
4. User clicks Login/Signup on React frontend
5. Firebase Auth handles authentication — Google OAuth or email/password
6. Firebase returns an ID token to the React app
7. React stores token and sends it in every API request as Authorization: Bearer {token}
8. Express middleware verifies token using Firebase Admin SDK on every protected route
9. If valid, middleware attaches user object to req.user and continues to route handler
10. If invalid, middleware returns 401 Unauthorized immediately
3.3 File Upload Flow
11. User selects PDF file in React frontend
12. React uses PDF.js to extract text from the PDF client-side
13. React sends both the file (FormData) and extracted text (JSON) to Express API
14. Express receives the file in memory using Multer (memory storage — no disk writes)
15. Express uploads file buffer to Cloudinary using upload_stream
16. Cloudinary returns a secure_url and public_id
17. Express saves secure_url and public_id to MongoDB Document model
18. Express sends extracted text to OpenRouter for AI analysis
19. AI result is saved to MongoDB Analysis model
20. Express returns full analysis result to React frontend

--- PAGE 6 ---
3.4 AI Analysis Flow
21. Express receives IEP text + child profile data
22. Express constructs the analysis prompt using the profile data as context
23. Express calls OpenRouter API with primary model (llama-4-maverick)
24. OpenRouter returns AI response as JSON string
25. Express parses JSON — if parsing fails, retries with regex extraction
26. If primary model fails, Express retries with backup model (gemini-2.0-flash)
27. Parsed result is validated — all required fields must be present
28. Valid result is saved to MongoDB and returned to frontend

7.4 AI Service Error Handling Rules
The OpenRouter service must follow these rules exactly:
29. Always try the primary model first: meta-llama/llama-4-maverick:free
30. If primary model returns an error, automatically retry with:
google/gemini-2.0-flash-exp:free
31. If backup model also fails, retry with: mistral/mistral-7b-instruct:free
32. Parse the AI response as JSON — if JSON.parse fails, use regex to extract the JSON
object
33. If all models fail after 3 total attempts, return a 503 error to the client
34. Cache successful AI responses in MongoDB — never call AI twice for the same
document
35. Always include the disclaimer field in every response before returning to frontend

12. Security Requirements
IEP Desk handles sensitive information about children with disabilities. Security is
non-negotiable. The following requirements must be implemented before any user data is
accepted.
12.1 Authentication Security
• All API routes except health check must require valid Firebase token
• Firebase Admin SDK must verify token on every request — never trust client-sent user
IDs
• Admin routes must check user.role === 'admin' after auth check
• Never store Firebase tokens in localStorage — use memory or httpOnly cookies
12.2 Data Security
• All MongoDB queries must filter by userId — users can only access their own data
• Before returning any document, profile, or analysis — verify it belongs to the requesting
user
• Cloudinary folders organized by userId — /iep-desk/{userId}/ — prevents access
confusion
• Never log or store raw IEP text beyond what is needed for the analysis cache
• Add FERPA disclaimer to all pages that handle student educational records
12.3 Input Validation
• Validate all incoming request bodies on the server — never trust client data
• File upload: only allow PDF, PNG, JPG, JPEG — reject all other file types
• File size limit: maximum 10MB per upload
• Text inputs: sanitize all strings before saving to MongoDB
• Rate limit all AI endpoints: maximum 10 requests per user per hour
12.4 Legal Disclaimers (Required on Every AI Output)
This exact disclaimer must appear on every AI-generated response:
"This analysis is for informational and educational purposes only.
It does not constitute legal advice and does not create an attorney-client relationship.
Always verify information against your original IEP document.
For complex legal disputes, consult a qualified special education attorney or advocate."
